gh-forgejo-shim python development testing loop

Test Python changes without reinstalling every time

During normal development, install the project once in editable mode, then run tests and CLI smoke checks against the live source tree. Reinstall only when packaging metadata, dependencies, or entry points change.

The Rule

Use python -m pip install -e . for development. Editable install keeps command entry points available while importing the package from the checkout, so ordinary .py edits are picked up the next time tests or commands run.

No Reinstall

Needed after normal source edits in package modules or tests. Re-run the command or test process instead.

Editable Reinstall

Useful after changing dependencies, package metadata, console scripts, or pyproject.toml.

Real Install

Use only when validating the package as a user would install it, including pipx or release packaging behavior.

One-Time Development Setup

Create a local virtual environment, activate it, then install the package in editable mode. If the repo defines optional test dependencies, use the test extra.

cd "$HOME/dev/gh-forgejo-shim"
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e ".[test]"

If the project does not define a test extra, install editable mode directly and add the test runner separately:

python -m pip install -e .
python -m pip install pytest

Daily Test Loop

Run the smallest check that proves the thing you changed, then broaden before committing.

python -m pytest

Run one test file while iterating:

python -m pytest tests/test_auth.py

Run one specific test when you know the failing behavior:

python -m pytest tests/test_auth.py::test_name_here

CLI Smoke Checks

After an editable install, console commands should point at the live checkout. Re-run the command after each edit; the process starts fresh and imports the latest source.

gfj --help
gh-forgejo-shim --help
gh-forgejo-shim doctor

You can also bypass installed entry points and run the module directly:

python -m gh_forgejo_shim --help

When To Reinstall

  • Do not reinstall after changing regular Python files such as auth.py, routing.py, or tests.
  • Reinstall editable after changing pyproject.toml, dependency declarations, package data, or console-script entry points.
  • Do a real install when validating pipx behavior, wheel contents, release packaging, or the installed wrapper path.
python -m pip install -e .
python -m pip install .

Pipx Validation

Use pipx when you need to prove the package behaves like a user-installed command-line tool. This is a slower packaging check, not the inner development loop.

pipx install --force .
gfj --help
gh-forgejo-shim doctor

Troubleshooting

  • If imports fail, confirm the virtual environment is active with which python.
  • If gfj still runs old behavior, confirm it resolves inside the virtual environment with which gfj.
  • If tests cannot find pytest, install the test extra or run python -m pip install pytest.
  • If GUI-launched tools still see old behavior, reinstall through the same mechanism those tools use and restart the app.
