Temporary Git Recorder Core

Added the isolated recorder core for creating a temporary git wrapper around one Codex launch session. The wrapper writes sanitized JSONL records and can be removed without changing persistent PATH configuration.

Summary

The new recorder module creates a managed temporary git executable, returns the launch environment a caller should use, invokes a caller-specified real git executable, forwards output, preserves the exit code, and appends one JSON record per invocation.

Changes Made

Recorder API

Added GitRecorder, create_git_recorder, remove_git_recorder, and git_recorder_script.

Launch Env

The return object includes wrapper_dir, wrapper_path, trace_path, real_git, path, and env.

Trace Safety

Trace records include timestamp, cwd, sanitized argv, duration, exit code, and stdout/stderr byte counts. Excerpts require FJ_SHIM_TRACE_BODY=1.

Unit Tests

Added fake-git coverage for wrapper creation, execution, cleanup, output forwarding, byte counts, and redaction.

Context

This task intentionally does not wire the recorder into cli.py. The main integration flow can call the helper, launch Codex with recorder.env or recorder.path, and call remove_git_recorder afterward.

Important Implementation Details

  • The generated wrapper is a standalone Python script built with stdlib-only code.
  • create_git_recorder accepts a fake real_git path, which keeps tests independent from the machine's installed git.
  • Output is forwarded through reader threads, allowing the wrapper to count bytes without storing full stdout or stderr bodies.
  • Secret-like argv values, authorization headers, token assignments, URL userinfo, and common token prefixes are redacted before records are appended.
  • Cleanup refuses unmanaged directories by checking the wrapper marker before deleting the temporary directory.

No CLI behavior changes yet. After integration, Codex launch sessions can temporarily shadow git, collect sanitized command telemetry, and cleanly remove the wrapper directory at the end of the session.

Issues, Limitations, and Mitigations

  • Not wired yet: callers must use the helper directly until the main integration adds CLI/session wiring.
  • Redaction is best-effort: excerpts are opt-in and sanitized, but the recorder is not a formal secret scanner.
  • Shared workspace: unrelated tracked and untracked work was present and intentionally left outside this commit.
