2026-05-31 Repository implementation Issue gh-forgejo-shim-ufb

Fixed Codex.app PR board compatibility for Forgejo pull requests

Codex.app now gets GitHub-shaped pull request data from the Forgejo shim for the PR board, current branch status, merged PR lookups, and PR checks probes.

Summary

Codex.app had been reaching Checking pull request and then falling back to Pull request status unavailable. The installed shim now returns parse-safe GitHub CLI-compatible data for the PR board path, and the user confirmed the Codex.app sidebar now resolves the Forgejo pull request.

Changes Made

Normalized board fields

Added deletions, numeric additions/deletions defaults, string mergeable states, uppercased mergeStateStatus, and merged-state detection.

Accepted merged lists

gh pr list --state merged now maps to Forgejo's closed API query and filters results to pulls that actually merged.

Handled checks probes

gh pr checks --json ... now returns an empty JSON list instead of failing, which is enough for Codex to treat the command as supported when Forgejo has no GitHub-style checks payload.

Context

The Codex app bundle showed that its native PR workflow asks gh pr list for board fields such as additions, deletions, mergeable, mergeStateStatus, and statusCheckRollup, including an additional merged PR query. Before this change, the shim returned missing or incompatible values for some of those fields and rejected --state merged.

The fix was tested against the real Islandflow Forgejo repository on branch lavender/all-quiet, where PR #16 is the current branch pull request.

Important Implementation Details

  • mergeable: false from Forgejo is normalized to UNKNOWN, because GitHub CLI consumers expect string enum-like values rather than booleans.
  • Missing diff counts are normalized to 0, so strict JSON schemas do not reject null values.
  • Merged PRs are detected from merged or merged_at, then emitted as MERGED.
  • The checks command is intentionally minimal and parse-safe. It does not attempt to emulate GitHub Actions data from Forgejo CI.

  • User confirmed Codex.app's PR workflow now works after the fix.

Issues, Limitations, and Mitigations

  • Forgejo checks data is not fully modeled. The shim returns [] for pr checks. This avoids Codex parse failures, but it does not display real Forgejo CI checks yet.
  • Mergeability remains conservative. Unknown Forgejo mergeability maps to UNKNOWN rather than pretending the PR is mergeable.
  • Codex.app may cache an old probe result. Reopening or refreshing the affected thread can be necessary after reinstalling the shim.

Follow-up Work

No required follow-up issue was filed. A useful future enhancement would be mapping Forgejo CI or status endpoints into GitHub-style gh pr checks rows if Codex.app starts displaying those details in the sidebar.
