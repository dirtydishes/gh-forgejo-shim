Map Forgejo CI Status Into GitHub-Shaped Output

Forgejo commit statuses now feed the shim's GitHub-compatible PR status surfaces, so tools that ask for statusCheckRollup or gh pr checks receive useful check data instead of an empty placeholder.

ScopeForgejo client, PR routing, status normalization, tests, README.
Issuegh-forgejo-shim-wk9
Validation74 unit tests passed with unittest; compileall passed.

Summary

Added commit-status support for Forgejo pull requests and translated those statuses into GitHub CLI-shaped check rows and rollup items.

Changes Made

  • Added ForgejoClient.list_commit_statuses for /statuses/{sha} responses.
  • Added normalizers for statusCheckRollup objects and gh pr checks --json rows.
  • Updated PR list, view, status, and checks routing to fetch statuses from the PR head SHA when needed.
  • Updated tests to verify rollup output, check row output, and repeated-status de-duplication by context.
  • Updated the README to document status/check JSON behavior.

Context

GitHub-oriented tools often probe gh pr list, gh pr status, or gh pr checks to decide whether a branch is ready. Before this turn, the shim accepted those probes but did not translate Forgejo CI/status data into the fields those tools expect.

Important Implementation Details

  • Status data is fetched only when statusCheckRollup is requested or when gh pr checks runs, keeping ordinary PR calls cheap.
  • Forgejo states are mapped into stable buckets: success becomes pass, failures and errors become fail, and everything else becomes pending.
  • When Forgejo returns repeated status updates for a context, the latest timestamp wins.
  • If a pull request has no head SHA, the shim returns an empty status/check list instead of failing.

def _normalize_status_check_rollup(value: Any) -> list[dict[str, Any]]:
    if not isinstance(value, list):
        return []

def _latest_statuses(statuses: list[dict[str, Any]]) -> list[dict[str, Any]]:
    latest: dict[str, dict[str, Any]] = {}
    for status in statuses:
        key = _status_name(status)
        existing = latest.get(key)
        if existing is None or _status_timestamp(status) >= _status_timestamp(existing):
            latest[key] = status
    return sorted(latest.values(), key=_status_name)

def _status_name(status: dict[str, Any]) -> str:
    context = status.get("context") or status.get("name") or status.get("title")
    return str(context) if context else "status"




def _status_workflow(status: dict[str, Any]) -> str:
    workflow = status.get("workflow") or status.get("workflow_name") or status.get("context")
    return str(workflow) if workflow else _status_name(status)




def _status_url(status: dict[str, Any]) -> str | None:
    url = status.get("target_url") or status.get("url") or status.get("html_url")
    return url if isinstance(url, str) and url else None




def _status_timestamp(status: dict[str, Any]) -> str:
    value = status.get("updated_at") or status.get("created_at")
    return value if isinstance(value, str) else ""




def _status_state(status: dict[str, Any]) -> str:
    raw = str(status.get("state") or status.get("status") or "").lower()
    if raw in {"success", "successful", "ok", "pass", "passed"}:
        return "SUCCESS"
    if raw in {"failure", "failed"}:
        return "FAILURE"
    if raw in {"error", "cancelled", "canceled", "warning"}:
        return "ERROR"
    return "PENDING"




def _status_conclusion(state: str) -> str | None:
    if state == "SUCCESS":
        return "success"
    if state == "FAILURE":
        return "failure"
    if state == "ERROR":
        return "failure"
    return None




def _status_bucket(state: str) -> str:
    if state == "SUCCESS":
        return "pass"
    if state in {"FAILURE", "ERROR"}:
        return "fail"
    return "pending"




def _check_state(state: str) -> str:
    if state in {"SUCCESS", "FAILURE", "ERROR"}:
        return "completed"
    return "pending"




def _status_completed_at(status: dict[str, Any], state: str) -> str | None:
    if state not in {"SUCCESS", "FAILURE", "ERROR"}:
        return None
    value = status.get("updated_at") or status.get("created_at")
    return value if isinstance(value, str) else None

Expected Impact for End-Users

Codex, T3, editors, and bots that expect GitHub-style PR checks should now see meaningful Forgejo status data for allowlisted repositories, including pass, fail, and pending check rows.

Issues, Limitations, and Mitigations

  • bd dolt pull failed because local and remote Dolt histories have no common ancestor. I left the local Beads issue in place and did not run destructive recovery.
  • The mapping uses Forgejo commit statuses. If a Forgejo instance exposes richer Actions jobs separately without commit statuses, those jobs will not appear until the instance publishes statuses or the shim adds a second API source.
  • The rollup shape is intentionally practical rather than a complete GitHub GraphQL object model.
