Surfaced Forgejo PR links in Codex.app

The shim now prints a Forgejo pull request URL that Codex.app can recognize after gh pr create, without changing the real Forgejo URL or JSON output contract.

Summary

Codex.app was creating Forgejo pull requests but not surfacing the resulting link. The local Codex.app bundle showed that its create-PR action scans command output for an HTTP URL containing GitHub-style /pull/7. Forgejo correctly returns /pulls/7, so the URL was skipped. This change appends a harmless fragment marker, such as #codex-pr=/pull/7, to default human output only.

Changes Made

Codex Link Marker

Added format_created_pull_url so default gh pr create output keeps the Forgejo /pulls/7 path and adds a Codex-recognized fragment marker.

Clean JSON Output

Left gh pr create --json ... unchanged, so API-style callers still receive the canonical Forgejo pull request URL.

Regression Test

Added a small parser copy of Codex.app's URL recognition rule and a focused test proving the shim output is now detectable.

README Note

Documented why the default URL includes a fragment marker and why it exists only for Codex.app compatibility.

Context

The important distinction is that the Forgejo URL was already valid for a browser, but Codex.app was filtering it out before presenting the link. The app's local create-PR action combines stdout and stderr, extracts HTTP URLs, trims trailing punctuation, and accepts only URLs that match /pull/\d+. A Forgejo URL like https://git.example.com/owner/repo/pulls/7 therefore failed the GUI's link detector even though it pointed to the right page.

Important Implementation Details

  • The fragment preserves browser behavior: opening https://host/owner/repo/pulls/7#codex-pr=/pull/7 still loads the real Forgejo pull request page.
  • The formatter is idempotent for URLs that already contain a GitHub-style /pull/7 path or fragment.
  • If the value is not a URL or does not contain a pull request number, the shim prints it unchanged rather than guessing.
  • The regression test mirrors the observed Codex.app recognition rule, so a future change that drops the marker will fail before reaching the installed wrapper.

+def format_created_pull_url(value: object) -> str:
+    if not isinstance(value, str) or not value:
+        return ""
+    if CODEX_CREATE_PR_URL_RE.search(value):
+        return value
+    number = _number_from_url(value, kind="pull")
+    if number is None:
+        return value
+    marker = f"codex-pr=/pull/{number}"
+    parsed = urllib.parse.urlparse(value)
+    if CODEX_CREATE_PR_URL_RE.search(parsed.fragment):
+        return value
+    fragment = f"{parsed.fragment}&{marker}" if parsed.fragment else marker
+    return urllib.parse.urlunparse(parsed._replace(fragment=fragment))
