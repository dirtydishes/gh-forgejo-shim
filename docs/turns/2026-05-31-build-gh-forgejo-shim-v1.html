Summary

Implemented V1 of gh-forgejo-shim: a Python 3.11+ package with no runtime dependencies, a durable management CLI, a reversible generated gh wrapper, opt-in Forgejo host routing, PR create/view/status behavior, GitHub-shaped JSON normalization, docs, and tests.

Context

Codex.app expects GitHub-like PR commands in repositories. Forgejo repositories do not speak the full GitHub CLI contract, so this project narrows the surface area to commands Codex is likely to call and leaves everything else with the real gh. This keeps GitHub behavior intact while making Forgejo repos less brittle under automation.

The repository uses Beads. The implementation issue is gh-forgejo-shim-uku. Two follow-up Beads tasks were opened for work that should remain outside this V1 pass.

Important Implementation Details

  • src/gh_forgejo_shim/routing.py decides whether to delegate or route to Forgejo.
  • src/gh_forgejo_shim/repo.py parses HTTPS, SSH, scp-style SSH, host-prefixed, and plain owner/repo specs.
  • src/gh_forgejo_shim/create.py translates supported gh pr create flags and rejects GitHub-only metadata flags.
  • src/gh_forgejo_shim/forgejo.py implements the minimal REST client using urllib.
  • src/gh_forgejo_shim/shim.py writes a marked wrapper and refuses unrelated files unless --force is passed.
  • src/gh_forgejo_shim/doctor.py checks executable discovery, PATH ordering, hosts, and auth.

Expected Impact for End-Users

Users can install the package with pipx, explicitly allowlist Forgejo hosts, and install the reversible gh wrapper. Codex.app can then run common PR create/status/view commands in Forgejo repositories without breaking normal GitHub repositories on the same machine.

Issues, Limitations, and Mitigations

  • V1 intentionally supports only gh pr create/new/status/view for Forgejo hosts; every other command delegates to real gh.
  • No native login command exists. Auth checks env tokens first, then best-effort local fj/tea/gitea config discovery.
  • Forgejo REST behavior is unit-tested with fakes, not yet exercised against a live Forgejo fixture.
  • --jq and broader GitHub CLI JSON behavior are not implemented in this V1 surface.
