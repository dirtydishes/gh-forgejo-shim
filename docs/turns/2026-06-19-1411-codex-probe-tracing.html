2026-06-19 Repository implementation Issue gh-forgejo-shim-1ii

Added Codex probe capture and replay diagnostics

The shim can now record opt-in gh and temporary git probe traces, summarize those JSONL files, and run a local Codex-style smoke sequence without changing default runtime behavior.

Summary

Implemented a diagnostic-first loop for the reported Codex.app states: CLI availability, PR status availability, and branch selector freshness. Tracing is enabled only by FJ_SHIM_TRACE, body excerpts require FJ_SHIM_TRACE_BODY=1, and auth-sensitive output is redacted or suppressed.

Changes Made

Shim Trace Capture

Added src/gh_forgejo_shim/trace.py and wired routing.run_gh() to append route, host/repo, duration, exit code, env keys, and output-size records when tracing is enabled.

Trace Summary

Added gfj trace summarize <trace.jsonl> to group failures, slow calls, unsupported commands, routes, and known Codex probe patterns.

Codex Smoke Replay

Added gfj trace smoke, which runs the observed read-only gh and raw git probe family and reports exit codes, timings, and output sizes.

Temporary Git Recorder

Added gfj trace git-recorder create/remove for one-session raw git tracing when branch UI behavior bypasses the gh shim.

Context

Previous fixes added Forgejo-shaped support for gh auth status, gh api user, gh pr status, and Codex PR board fields, but the next compatibility step needed exact observations instead of more guessing. This change turns live Codex subprocess behavior into replayable records and focused tests.

Important Implementation Details

  • run_gh() remains a no-op for tracing unless FJ_SHIM_TRACE is present.
  • Forgejo-routed commands use forwarding capture streams, so terminal output still appears while byte counts and optional excerpts are recorded.
  • Delegated gh calls use a traced subprocess path only when tracing is enabled; the normal delegate path still calls the existing run_program().
  • The temporary git recorder prepends one wrapper directory to PATH, streams real git output, records JSONL, and removes the directory with gfj trace git-recorder remove.
  • Documentation was updated in README.md and docs/dev/testing.html so future Codex diagnosis starts from traces and smoke checks.

Expected Impact for End-Users

Users and future agents can capture the exact command that caused a Codex.app compatibility state, summarize the likely failure class, and replay the local probe family before changing shim behavior. The default installed shim remains unchanged until tracing is explicitly enabled.

Issues, Limitations, and Mitigations

  • Limitation: I did not launch the GUI Codex.app reproduction flow in this pass. The new trace and smoke commands are the intended path for that manual validation.
  • Tradeoff: Traced delegate gh calls capture subprocess output before forwarding it. This is opt-in only and keeps the existing streaming delegate path for normal runtime.
  • Risk: Body excerpts can still contain surprising local paths or command text, so they remain disabled unless FJ_SHIM_TRACE_BODY=1 is set. Tokens and auth-sensitive patterns are redacted, and gh auth token stdout excerpts are suppressed.
