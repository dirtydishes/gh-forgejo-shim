Keep GitHub CLI Available After Shim Uninstall

Repaired the macOS uninstall path so removing the Forgejo gh shim does not leave Codex.app or other GUI-launched tools unable to find the real GitHub CLI.

Summary

Uninstalling the shim on macOS now keeps GUI apps on a usable PATH. If the shim-owned LaunchAgent exists, uninstall-shim rewrites it to a Homebrew/system PATH that excludes ~/.local/bin, then applies that value to the current launchd user session.

Changes Made

  • Added default_real_gh_gui_path() for a fallback PATH that can find Homebrew, MacPorts, and system tools without preferring the removed shim directory.
  • Changed uninstall-gui-path to remove the LaunchAgent and immediately apply the real-gh fallback PATH for newly launched GUI apps.
  • Changed uninstall-shim on macOS to rewrite an existing shim-owned GUI PATH helper to the real-gh fallback.
  • Updated rollback/configuration docs to explain the Codex.app restart behavior.
  • Added regression coverage for direct GUI helper removal and the full CLI uninstall-shim path.

Context

The user reported that after uninstalling the shim and restarting Codex, the app showed GitHub CLI unavailable. That can happen when macOS GUI launchd PATH no longer points at Homebrew's real gh. Removing the shim should not make GitHub tooling disappear; it should fall back to the real GitHub CLI.

Important Implementation Details

  • The normal install path still prefers ~/.local/bin so the shim can intercept allowlisted Forgejo repositories.
  • The uninstall fallback deliberately excludes ~/.local/bin so GUI apps do not probe a removed wrapper first.
  • The repair only rewrites the macOS GUI PATH helper when the shim-owned LaunchAgent already exists.
  • CLI output now prints the PATH it applied, making future diagnosis less guessy.

Expected Impact for End-Users

Codex.app and other GUI-launched tools should keep finding the real GitHub CLI after the shim is removed, as long as gh is installed in one of the common Homebrew, MacPorts, or system locations.

Validation

  • PYTHONPATH=src python3 -m unittest tests.test_gui_path tests.test_cli: passed, 5 tests.
  • PYTHONPATH=src python3 -m unittest: passed, 92 tests.
  • /bin/launchctl getenv PATH on the local machine reports the real-gh fallback PATH.

Issues, Limitations, and Mitigations

  • The fallback only helps if the real gh exists in a common directory such as /opt/homebrew/bin or /usr/local/bin. Users with custom install paths can still run install-gui-path --path ....
  • Existing GUI app processes keep their old environment until they restart. The docs and CLI output now say to restart Codex.app after the repair.
