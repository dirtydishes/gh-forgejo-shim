Add Forgejo HTTP Fixture Tests

Added a stdlib-only integration layer that drives ForgejoClient through a local HTTP server, covering real request construction without Docker or a live Forgejo instance.

Summary

The client can now be pointed at a local base URL for tests while preserving the default https://host/api/v1 behavior in normal use. A new integration test module verifies HTTP methods, URL quoting, headers, JSON bodies, query strings, text diffs, response shape handling, invalid JSON, and HTTP errors.

Changes Made

  • Added optional base_url support to ForgejoClient.
  • Routed all repo API URLs through a shared helper so localhost fixtures and production hosts use the same path construction.
  • Added tests/test_forgejo_integration.py with a ThreadingHTTPServer fixture that records requests and serves Forgejo-shaped responses.
  • Closed HTTPError objects after reading error bodies to avoid resource warnings on modern Python.

Context

The previous test suite covered routing and normalization with fake client objects. This new layer sits one boundary lower: it uses real urllib.request calls against a local server, so regressions in paths, headers, encoded query strings, or JSON/text decoding are visible without needing a disposable Forgejo container.

Important Implementation Details

Base URL override: ForgejoClient(token, base_url="http://127.0.0.1:PORT") now builds URLs as {base_url}/api/v1/.... Without an override, URLs are still built from the repo host as before.

Fixture behavior: the test server records method, path, parsed query, headers, raw body, and decoded JSON body for each request. Tests assert those records directly.

Offline scope: the fixture validates Forgejo-compatible HTTP boundary behavior, not full real-server parity.

Expected Impact for End-Users

End users should see no behavior change. The main benefit is maintainability: future client changes have a faster offline safety net that catches broken Forgejo REST calls before they reach real repositories.

Validation

  • Ran PYTHONPATH=src python3 -m unittest.
  • Result: Ran 102 tests in 10.747s, OK.
  • Confirmed the warning-free HTTP error path after explicitly closing HTTPError.

Issues, Limitations, and Mitigations

  • The fixture is Forgejo-compatible but intentionally not a full Forgejo implementation.
  • The tests do not cover TLS, auth storage, server pagination, or network retries.
  • The local server keeps the integration layer fast and deterministic, which is the intended first pass.
