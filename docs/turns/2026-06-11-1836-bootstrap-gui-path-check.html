Bootstrap Catches macOS GUI PATH Gaps

Updated gfj bootstrap so it reports when newly opened Codex.app windows cannot see the installed shim through launchd PATH.

ScopeBootstrap diagnostics, README setup guidance, and regression tests.
Issuegh-forgejo-shim-dge
Validation94 unit tests passed.

Summary

gfj bootstrap now checks the macOS user launchd PATH, not only the shell PATH. If launchd cannot see the shim directory, bootstrap returns a fix check and prints gh-forgejo-shim install-gui-path.

Changes Made

  • Added a bootstrap-time macOS GUI PATH check on Darwin.
  • Added injectable launchd_path and check_gui_path parameters for deterministic tests.
  • Updated README quickstart text so users run install-gui-path when bootstrap reports the GUI gap.
  • Added tests for missing and present GUI shim PATH.

Context

The live failure came from running gfj bootstrap after uninstalling the shim while launchd PATH still excluded ~/.local/bin. The shell setup looked healthy, but Codex.app could not route Forgejo repository probes through the shim after restart.

Important Implementation Details

  • The check uses current_launchd_path() and path_contains_dir(), matching the existing doctor behavior.
  • Non-macOS bootstrap behavior is unchanged.
  • Bootstrap prints the repair command instead of silently changing GUI PATH, preserving explicit user setup.

Expected Impact for End-Users

After reinstalling the shim, users should get a clear bootstrap warning before restarting Codex into a state where Forgejo pull request status is unavailable.

Validation

Issues, Limitations, and Mitigations

  • Existing Codex.app processes still need a restart after install-gui-path; bootstrap can only diagnose the launchd environment for newly opened apps.
  • A source-tree bootstrap smoke test briefly allowlisted github.com in the user config; that entry was removed immediately and the config now lists only the Forgejo hosts.

Follow-up Work

No new follow-up issue is required. Existing release work can carry this diagnostic improvement into the installed package.
